## Architecture

TestWright is an AI-powered test case generator built on a **LangGraph sequential multi-agent pipeline**. A functional specification (Markdown) enters the pipeline and structured test cases (JSON + Markdown + PNG) come out.

### Pipeline Flow (11 nodes in sequence)

```
Functional Spec (Markdown)
  ā†’ Parser ā†’ Navigation ā†’ Chunker ā†’ Summary
  ā†’ Test Generator ā†’ Assembler
  ā†’ Verify Flag ā†’ Verify Ideal ā†’ Verify Matcher ā†’ Execution Planner
  ā†’ Finalize ā†’ JSON + Markdown + PNG output
```

State flows through all nodes via `PipelineState` (a TypedDict defined in [core/state.py](core/state.py)).

### Key architectural layers

| Layer | Location | Role |
|-------|----------|------|
| Orchestration | `core/generator.py` | Entry point; compiles LangGraph and invokes the pipeline |
| Graph topology | `core/graph.py` | Wires 11 nodes in sequence |
| Node functions | `core/nodes.py` | Each node instantiates its agent and transitions state |
| Agents | `agents/` | One file per agent; all inherit `BaseAgent` in `agents/base.py` |
| Data models | `models/schemas.py`, `models/enums.py` | Dataclasses and enums shared across the pipeline |
| Exporters | `exporters/` | JSON and Markdown serialization after the pipeline finishes |
| CLI | `cli.py` | `--generate` and `export-md` subcommands |

### Agent responsibilities

- **Parser**: Parses functional spec into `ParsedModule` objects (modules, workflows, rules).
- **Navigation**: Builds a directed page graph (`NavigationGraph`) and renders a PNG.
- **Chunker**: Splits modules into workflow-based `WorkflowChunk` objects.
- **Summary**: Generates concise module summaries capturing verify/action states.
- **Test Generator**: Produces positive, negative, and edge-case `TestCase` objects per chunk.
- **Assembler**: Deduplicates tests, assigns `MODULE-NNN` IDs, links to the navigation graph.
- **Verify Flag**: Flags tests that modify persistent state (requires post-action verification).
- **Verify Ideal**: Designs ideal verification scenarios for each flagged test.
- **Verify Matcher**: RAG-based embedding search to match ideal verifications to real tests.
- **Execution Planner**: Compiles PRE ā†’ ACTION ā†’ POST execution sequences.
- **Finalize**: Validates output, writes `test-cases.json` and `navigation_graph.png`.

### LLM provider configuration

`BaseAgent` uses `httpx` to call any OpenAI-compatible endpoint. Provider is selected via `--provider`:

- `openai` ā†’ `https://api.openai.com/v1`
- `github` ā†’ `https://models.inference.ai.azure.com`
- `openrouter` ā†’ `https://openrouter.ai/api/v1`

### Input format

A directory passed to `--input` should contain:
- `functional_specification.md` (required) ā€” H2 headings per module, workflows, fields, rules
- `navigation.md` (optional) ā€” page relationships
- `mock_data.md` (optional) ā€” test credentials and sample data

See `examples/parabank/` and `examples/moodle/` for reference.

### Output format

- `test-cases.json` ā€” `TestSuiteOutput` serialized: test cases, navigation graph, execution plans, summary stats
- `navigation_graph.png` ā€” directed page graph visualization
- `test-cases.md` ā€” human-readable table (generated by `export-md`, not the main pipeline)
