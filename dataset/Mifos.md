# Mifos

## Website URL

<http://localhost:4200>

Tenant — default | Username — mifos | Password — password

## Navigation

Mifos is a microfinance and core banking platform built on Apache Fineract using Material Design layout with a left icon-based sidebar and persistent top navigation. After login, the top navigation bar displays the Mifos logo on the left and menu items: Institution, Accounting, Reports, Admin, Self Service, and Configuration Wizard, along with icons for global search, language selection, notifications, and user profile on the right. The Institution menu contains Clients, Groups, Centers, Accounting, Reports (with sub-items: All, Clients, Loans, Savings, Funds, Accounting, XBRL), Admin, and Self Service. The left sidebar provides icon-based shortcuts to commonly used sections (Clients, Groups, Centers, Products, and more) that navigate to corresponding pages. The Products hub (at /products) is a landing page listing all product categories: Loan Products, Savings Products, Share Products, Charges, Collateral Management, Delinquency Buckets, Products Mix, Fixed Deposit Products, Recurring Deposit Products, Manage Tax Configurations, and Floating Rates. Breadcrumbs at the top of the content area show the navigation path. Entity listing pages display data in Material Design tables with search, filter, sort, and pagination controls. Detail pages display entity information in card layouts with action buttons for state transitions. Forms use Material Design input fields with inline validation.

## Functional Description

### 1. Login

The login page contains Tenant (dropdown defaulting to "default"), Username, Password fields, and Login button. Valid credentials authenticate the user and redirect to Home. Invalid credentials, empty fields, or unrecognized tenant display error messages. Required fields show inline validation errors. Special characters are handled consistently. Sessions persist on page refresh. Two-factor authentication can be enabled at system or user level; when enabled, successful primary authentication prompts for second-factor token (OTP via email, SMS, or authenticator app); invalid TFA token denies access.

### 2. Home

The Home page is the main dashboard after login, displaying a welcome card with the user's name. The top navigation bar provides access to Institution, Accounting, Reports, Admin, Self Service, and Configuration Wizard modules with icons for global search, language selection, notifications, and user profile. The left sidebar shows icon-based shortcuts to commonly used sections (Clients, Groups, Centers, Products, etc.). A Search Activity field allows users to query recent system activities. Navigation adapts based on user roles and permissions. Sessions remain valid until explicit logout.

### 3. Dashboard

The Dashboard displays a Client Trends chart showing client growth (New Clients and Closed Clients) filtered by selected office. Below the chart are summary cards showing Amount Pending/Disbursed and Amount Collected metrics. A Search Activity field at the top queries recent system activities. The page displays "No Data" if no information is available for the selected office.

### 4. Global Search

Global Search is accessible from the search icon in the top navigation bar and works from any page after login. Users enter keywords (client names, account numbers, IDs) and the system searches Clients, Groups, Loans, and Savings accounts in real-time. Matching results appear in a dropdown grouped by entity type showing name, identifier, and status. Clicking a result navigates to that entity's detail page. Search supports partial matching and is case-insensitive; no matches display "No results found".

### 5. Client Management

The Clients page displays a table with columns for Name, Account Number, External ID, Status (Pending, Active, Closed, Rejected, Withdrawn), Office, and Staff. Users can search by name, account number, external ID, or mobile number and filter by status. The Create Client form collects Office (required), First Name (required), Last Name (required), Submitted On date, and optional fields like Mobile, Email, External ID, with Address Details, Family Members, and Identifiers (Document Type, Document Key) tabs. Duplicate External IDs are rejected. The Client Detail page shows client info with action buttons varying by status: Pending clients can be Activated, Rejected, or Withdrawn; Active clients can be Edited, Transferred, or Closed (only if no active accounts); Closed clients can be Reactivated. The detail page includes tabs for General, Accounts (Loans, Savings, Shares, Deposits), Identifiers, Family Members, Notes, and Documents.

### 6. Group Management

The Groups page displays a table with columns for Group Name, Account Number, Office, Status (Active, Pending, Closed), and Staff. The Create Group button opens a form to collect Name (required), Office (required), Staff, Submitted On date, Active status, External ID, and ability to add existing clients. The Bulk Import Groups feature allows downloading an Excel template and uploading files; import history shows name, time, record counts, and success/failure. The Group Detail page shows group info with action buttons: Activate, Edit, Close, Assign Staff, Transfer Clients. Tabs show Members, Loan Accounts (group loans and GLIM), Savings Accounts (group and GSIM savings), Notes, and Calendar for scheduling meetings. The Collection Sheet generates a batch entry sheet with loan repayment and savings deposit amounts.

### 7. Center Management

The Centers page displays a hierarchical table with columns for Name, Account Number, External ID, Status, and Office Name. The Create Center button collects Name (required), Parent Office (required), Opened On date (required, not future-dated), and External ID (optional). The system validates hierarchy to prevent circular relationships. Bulk Import Centers allows template download and file upload with import history tracking. The Center Detail page shows center info with action buttons: Activate, Edit, Close, Assign Staff. Tabs display Groups, Loan Accounts, Savings Accounts, Notes, and Calendar for scheduling. The Collection Sheet generates a batch entry sheet with all groups and clients showing loan repayment and savings deposit amounts.

### 8. Loan Products

The Loan Products page displays a table with columns for Name, Short Name, Expiry Date, and Status, plus a filter bar and Create Loan Product button. The 6-step wizard collects: Details (Product Name, Short Name, External ID, Fund, Start/Close Date, Description), Currency (selection, Decimal Places, Principal Amount bounds), Settings (Amortization method, Interest Method/Calculation Period, Repayment Strategy, Days in Year/Month, grace periods), Terms (Number of Repayments bounds, Repaid Every frequency, Interest Rate bounds), Charges (search-and-add interface), and Accounting (None/Cash-based/Accrual selection with GL account mappings for Fund Source, Loan Portfolio, Interest/Fees/Penalties Income, Losses Written Off, Overpayment Liability). All required fields show validation errors. Existing products can be viewed and edited.

### 9. Savings Products

The Savings Products page displays a table with columns for Name and Short Name, plus a filter bar and Create Savings Product button. The 6-step wizard collects: Details (Product Name, Short Name, Description, External ID), Currency (selection, Decimal Places, Multiples Of), Terms (Nominal Annual Interest Rate, Compounding/Posting Periods, Interest Calculation method, Days in Year), Settings (Minimum Opening Balance, Lock-in Period, Withdrawal Fee for Transfers, Minimum Balance for Calculation, Enforce Minimum Balance, Overdraft allowance with Max Amount/Rate, Withhold Tax with Tax Group, Dormancy Tracking with inactive/dormancy/escheat thresholds), Charges (search-and-add), and Accounting (None or Cash-based with GL mappings). Fixed Deposit Products include Pre-Closure settings, Deposit Term bounds, and Interest Rate Chart with tiered rates. Recurring Deposits include Mandatory Deposit Amount, Mandatory/Withdrawal checkboxes, Advance adjustment, and Recurring Frequency. All required fields show validation errors. Existing products display in detail view with Edit option.

### 10. Share Products

The Share Products page displays a table of all configured share products with columns for Product Name, Short Name, and Total Shares, plus a Create Share Product button. The 7-step wizard collects: Details (Product Name, Short Name, Description), Currency (selection, Decimal Places, Multiples Of), Terms (Total Number of Shares, Shares to Issue, Nominal/Unit Price, auto-calculated Capital Value), Settings (Allow Dividends for Inactive Clients checkbox, Minimum/Maximum/Nominal Shares per Client, Minimum Active Period, Lock-in Period), Market Price (table with From Date and Share Value per row), Charges (search-and-add interface), and Accounting (None or Cash-based with GL mappings for Share Reference, Share Suspense, Equity in Shares, Income from Fees, Share Equity). All required fields show validation errors. Existing products display in detail view with Edit and Delete options.

### 11. Charges

The Charges page displays a table of all charge definitions with columns for Name, Charge Applies To, Is Penalty, Is Active, and Is Paid Derived. The Create Charge form collects Charge Name (required), Charge Applies To (Loan, Savings Account, Client, or Shares), Currency (required), Charge Time Type (varies: Loans include Disbursement, Specified Due Date, Installment Fee, Overdue Fees, Tranche Disbursement; Savings include Specified Due Date, Savings Activation, Withdrawal Fee, Annual Fee, Monthly Fee, Overdraft Fee), Charge Calculation Type (Flat, Percentage of Amount, Percentage of Interest, Percentage of Total Outstanding), Amount (required), Is Penalty checkbox, Is Active checkbox, Tax Group (dropdown), and Payment Mode (Regular or Account Transfer). All required fields show validation errors. Clicking Submit creates the charge, which can then be associated with loan or savings products. Existing charges display in detail view with Edit and Delete options.

### 12. Floating Rates

The Floating Rates page displays a table with columns for Floating Rate Name, Is Base Lending Rate indicator, Is Active indicator, and Created By. The Create Floating Rate form collects Floating Rate Name (required), Is Base Lending Rate checkbox (only one base rate can exist), Is Active checkbox, and Rate Periods table with From Date, Interest Rate (percentage), and Is Differential Rate checkbox (if checked, rate is added to or subtracted from base lending rate). Multiple rows define rate changes over time. When a loan product is linked to a floating rate, the loan's interest rate adjusts automatically based on the applicable rate period. The detail view shows full rate history with Edit option to add new periods.

### 13. Delinquency Management

The Delinquency section provides Delinquency Ranges and Delinquency Buckets pages. The Delinquency Ranges page displays a table with Classification, Minimum Age Days, and Maximum Age Days. The Create Delinquency Range form collects Classification (required), Minimum Age Days (required), and Maximum Age Days (optional, if blank applies to all days beyond minimum). The Delinquency Buckets page lists all buckets with associated ranges. The Create Delinquency Bucket form collects Bucket Name (required) and interface to add multiple delinquency ranges in sequence (e.g., 1-29 days = Range 1, 30-59 days = Range 2, 60+ days = Range 3). Buckets are linked to Loan Products during configuration. When a loan becomes overdue, the system automatically classifies it into the appropriate range based on days past due, used for portfolio-at-risk reporting, provisioning calculations, and collection triggers. Existing buckets or ranges display in detail view with Edit and Delete options.

### 14. Loan Account

New loan applications are initiated from the Client Detail page. The Loan Application form is a multi-step wizard: Step 1 (Details) collects Product Name (dropdown pre-populating defaults), Loan Officer, Loan Purpose, Fund, Submitted On date, Expected Disbursement Date, Principal (bounded by product min/max), Number of Repayments, Repaid Every frequency/unit, Interest Rate (bounded by product min/max), and External ID. Step 2 (Terms) includes Repayment Strategy, Amortization, Interest Method, and Interest Calculation Period (pre-filled but adjustable) plus grace periods. Step 3 (Charges) shows inherited charges and allows adding additional ones. Step 4 (Collateral, optional) allows adding Collateral Type, Value, and Description. Clicking Submit creates the loan in Submitted and Pending Approval status. The Loan Detail page displays account number, product, client name, status badge, and balance. Status badges are color-coded: Pending Approval (yellow), Approved (blue), Active (green), Closed (gray), Overpaid (purple), Written Off (red). Action buttons vary by status: Pending Approval loans have Approve, Reject, Withdraw, Delete; Approved loans have Disburse, Undo Approval; Active loans have Make Repayment, Waive Interest, Write Off, Close, Reschedule, Prepay, Foreclosure, Charge Off, Assign Loan Officer. Tabs show Summary, Repayment Schedule, Transactions, Charges, Collateral, Notes, Documents.

### 15. Savings Account

New savings accounts are created from the Client Detail page. The form collects Product Name (dropdown pre-populating defaults), Field Officer, Submitted On date, Nominal Annual Interest Rate (pre-filled), Interest Compounding Period, Interest Posting Period, Interest Calculated Using, Days in Year, Minimum Opening Balance, Lock-in Period, and Allow Overdraft checkbox. The Charges section lists product charges and allows adding additional ones. Clicking Submit creates the account in Submitted and Pending Approval status. The Savings Account Detail page displays account number, product, client name, status badge, account balance, and available balance. Status badges include Submitted/Pending Approval, Approved, Active, Dormant, Closed, Blocked. Pending accounts have Approve, Reject, Withdraw Application options. Approved accounts have Activate, Undo Approval. Active accounts have Deposit, Withdraw, Post Interest, Calculate Interest, Close, Block Account/Debit/Credit. Tabs show Summary, Transactions, Charges, Documents.

### 16. Share Account

New share accounts are created from the Client Detail page. The Share Account Application form collects Share Product (dropdown pre-populating defaults), Submitted On date, Requested Shares (number bounded by product min/max per client), Application Date, Savings Account for Charges (dropdown to debit fees), and External ID (optional). The Charges section lists product-inherited charges and allows adding additional ones. Clicking Submit creates the account in Submitted and Pending Approval status. The Share Account Detail page displays account number, product, client name, status badge, total approved shares, total pending shares, and unit price. Pending accounts have Approve/Reject options. Approved accounts have Activate, Undo Approval. Active accounts have Apply Additional Shares, Redeem Shares (redemption amount = shares × unit price, credited to linked savings account), Close. Tabs show Purchased Shares, Dividends, Charges.

### 17. Fixed & Recurring Deposit Accounts

Fixed Deposit (FD) accounts are created from the Client Detail page by collecting Fixed Deposit Product (dropdown), Deposit Amount (numeric), Deposit Period (numeric and unit: Days, Months, Years), and Maturity Instructions (Transfer to Savings, Re-Invest, or Re-Invest Principal and Transfer Interest). The FD Detail page shows deposit amount, maturity date, maturity amount, interest rate, and status with buttons: Approve, Activate, Premature Close, Close on Maturity. Recurring Deposit (RD) accounts are created by collecting Recurring Deposit Product (dropdown), Mandatory Deposit Amount (numeric, per installment), Deposit Period, Deposit Frequency (Daily, Weekly, Monthly, Quarterly, Yearly), and Expected First Deposit On (date). The RD Detail page shows deposit schedule, total deposits made, maturity details, interest rate, and status with buttons: Approve, Activate, Deposit, Premature Close, Close on Maturity. Both FD and RD pages include tabs for Summary (account and interest details), Transactions (deposits and interest postings), and Charges. Interest rates are determined by Interest Rate Chart configured on product with tiered rates based on deposit amount and term.

### 18. Accounting - Chart of Accounts

The Chart of Accounts page displays a hierarchical tree or table of all General Ledger (GL) accounts organized by account type: Assets (blue), Liabilities (red), Equity (green), Income (purple), Expenses (orange). Each row shows GL Code, Account Name, Account Type, Manual Entries Allowed indicator, Usage (Header or Detail), and Description. Header accounts group parents; Detail accounts are used for actual postings. The Create GL Account form collects Account Type (required), Parent Account (required, must be header of same type), GL Code (required, must be unique), Account Name (required), Account Usage (Header or Detail), Manual Entries Allowed checkbox, Description (textarea), and Tag (dropdown). All required fields show validation errors. Clicking Submit creates the account. Existing accounts display in detail view with Edit and Delete options. GL Code must be unique within type, parent must match type and be header, header accounts cannot have direct postings, detail accounts are used for posting, and account deletion is prevented if already used in transactions or mappings.

### 19. Accounting - Journal Entries

The Journal Entries page displays a data table of recent entries with columns for Entry ID, Office, Transaction Date, Created Date, Transaction ID, Type (Debit/Credit), GL Account, and Amount. A filter bar allows filtering by Office, GL Account, Date Range, Transaction ID, and Entry Type. The Create Journal Entry form collects Office (required), Currency (required), Reference Number, Transaction Date (required), Payment Type and details (Account Number, Check Number, Receipt Number, Routing Code, Bank Number), Entry lines (GL Account with search/autocomplete, Amount) with Add Row, and Comments. A running total shows debit, credit, and difference. Total debits must equal total credits; unbalanced entries are rejected. Debit and credit amounts must match. Journal entries cannot be posted in closed accounting periods; GL accounts must have manual entry flag enabled.

### 20. Accounting Rules & Financial Activity Mappings

The Accounting Rules page displays a data table of all configured accounting rules with columns for Rule Name, Office, Debit Account(s), and Credit Account(s). The Create Accounting Rule form collects Office (dropdown for scope, blank applies to all), Rule Name (required), Debit Tags/Debit Account (dropdown or multi-select listing GL accounts to debit), Allow Multiple Debit Entries checkbox, Credit Tags/Credit Account (dropdown or multi-select), and Allow Multiple Credit Entries checkbox. Clicking Submit creates the rule. Existing rules display in detail view with Edit and Delete options. The Financial Activity Mappings page displays a table mapping financial activities to GL accounts, each showing Financial Activity (Asset Transfer, Liability Transfer, Cash at Mainvault, Opening Balances Transfer, Payable Dividends, Fund Source, etc.) and mapped GL Account. The Create Mapping form collects Financial Activity (dropdown of unmapped activities) and GL Account (dropdown). Each financial activity can only be mapped once. These mappings enable automatic posting of accounting entries for internal transfers, teller operations, and system-level financial activities.

### 21. Provisioning

The Provisioning section is accessed from the Accounting menu. The Provisioning Criteria page displays a table with Criteria Name and Created Date. The Create Provisioning Criteria form collects Criteria Name (required) and Definitions table where each row specifies Loan Product (dropdown), Category (STANDARD/SUB-STANDARD/DOUBTFUL/LOSS based on delinquency), Minimum Age (minimum overdue days), Maximum Age (maximum overdue days), Provisioning Percentage (percentage of outstanding principal), Liability Account (GL account dropdown), and Expense Account (GL account dropdown). Multiple rows cover different overdue ranges and products. The Provisioning Entries page lists all generated entries with Entry Date, Journal Entry Created, Created By, and action buttons. The Create Provisioning Entry button generates new entries based on current loan portfolio and configured criteria; the system calculates required provision amount for each loan category and creates corresponding journal entries. Each entry can be reviewed showing breakdown by product and category, or recreated.

### 22. Users & Roles

The Users page displays a data table of all users with columns for Username, Email, Office, Assigned Roles, Status (Active, Inactive), and action buttons. The Create User form collects Username (required, unique), First Name (required), Last Name (required), Email (optional), Office (required), Password (required, 8+ characters with mixed case/numbers), Confirm Password, and Select Roles (multi-select). Duplicate usernames are rejected. The User Detail page displays user info with Edit option and role management buttons. The Roles page displays all roles with Role Name, Description, Is System Defined (cannot delete), and Status (Active/Inactive). The Create Role form collects Role Name (required, unique), Description (optional), and Permissions table organized by module (Institution, Accounting, Reports, Admin, Self Service, Organization) with checkboxes for permissions (Client Create/Read/Update/Delete, Loan Create/Approve, Journal Entry Create, User Create, Role Create, etc.). Permissions can be assigned separately (Create, Approve, Reject) to enforce segregation of duties. The Role Detail page displays role with Edit option. Multiple users can be assigned to roles; permission precedence follows union (any role grants access). Role-based access control enforces dynamic navigation filtering, entity visibility scoped by office, action buttons visible only with required permission, maker-checker workflows separating create from approve/reject, and backend API endpoints enforcing role/office scope. Password management includes reset workflows (Forgot Password link and admin reset) with configurable expiration and history prevention. Session management includes configurable timeout (30-60 minutes inactivity), optional single session enforcement, and logout clears token. Two-factor authentication (TFA) can be enforced; when enabled, successful primary authentication prompts for second factor (OTP via email/SMS/authenticator), and invalid TFA denies access.

### 23. Offices

The Offices page displays a hierarchical tree or table of all offices with columns for Office Name, External ID, Parent Office, Opening Date, and hierarchy position. The Create Office form collects Office Name (required), Parent Office (dropdown listing offices for building hierarchy, required unless creating Head Office as root), Opened On Date (required), and External ID (optional). All required fields show validation errors. Clicking Submit creates the office under selected parent in organizational hierarchy. The Office Detail page shows office info with Edit option. Offices define the branch structure; clients, groups, staff, and accounting operations associate with specific offices. Office hierarchy enforces parent-child relationships and prevents circular dependencies.

### 24. Employees

The Employees page displays a data table of all staff with columns for Name, Office, Is Loan Officer indicator, and Status (Active, Inactive). The Create Employee form collects Office (required, determines home office and default scope), First Name (required), Last Name (required), Is Loan Officer checkbox (when enabled, staff appears in Loan Officer assignment dropdowns), Mobile Number (optional), Is Active checkbox (defaults to true; unchecked makes unavailable for new assignment), Joining Date (date picker), and External ID (optional). All required fields show validation errors. The Staff Detail page displays employee info with Edit option, showing office assignment, loan officer status, and activity status. Staff marked as Loan Officer appear in Loan Officer dropdowns when creating loans and can be assigned to clients, groups, centers for portfolio management. Staff assigned as loan/field officer become visible in assignment workflows. Staff can be transferred between offices via office edit. Inactive staff cannot be newly assigned but historical assignments remain visible. The system prevents deletion of staff referenced in historical transactions or active assignments.

### 25. Teller & Cashier Management

The Tellers page displays a data table of all configured tellers with columns for Teller Name, Office, Status (Active, Inactive), and Description. The Create Teller form collects Office (required), Teller Name (required), Description (textarea), Start Date (date picker, required), End Date (date picker, optional), and Status (Active/Inactive). A Cashiers section lists all cashier assignments with columns for Cashier Name, Start Date, End Date, Is Full Day, and Status. The Allocate Cashier form collects Staff (dropdown of employees, required), Start Date (required), End Date, Is Full Day checkbox, and Description. The Cashier Detail page shows allocated cash with Opening Balance and Cash In Hand. Available actions are Allocate Cash (add from vault) and Settle Cash (return to vault with Amount, Currency, Transaction Date). Cashier Transactions list shows all allocations, settlements, deposits, withdrawals with Date, Type, Amount, Running Balance.

### 26. Reports

The Reports section is accessed from the Reports menu in the top navigation bar. The page lists available reports grouped by category: All, Clients, Loans, Savings, Funds, Accounting, and XBRL. Each report shows Name, Type (Table, Pentaho, Chart), and Category. Clicking a report opens a parameters form with fields such as Office, Branch, Currency, Loan Product, Date Range (From/To pickers), Loan Officer, Fund, and other report-specific parameters. Required parameter fields show validation errors if empty. A Run Report button generates the report, which displays as a data table with sorting and pagination. Report output options include viewing as table on screen, exporting to Excel (XLS/XLSX), CSV, and PDF. Common reports include Loans Awaiting Disbursal, Loans Pending Approval, Active Loans Summary/Details, Obligation Met Loans Details, Portfolio at Risk, and Portfolio at Risk by Branch.

### 27. Account Transfers & Standing Instructions

The Account Transfers function is accessible from the Savings Account Detail page or the Self Service menu. The transfer form collects From Office (dropdown), From Client (dropdown), From Account Type (Savings Account or Loan Account), From Account (dropdown of client's accounts), To Office (dropdown), To Client (dropdown), To Account Type, To Account (can be same or different client), Transfer Amount (numeric, required), Transfer Date (date picker, required), and Description (textarea). If transfer amount exceeds available balance, an error is shown. Clicking Submit processes the transfer, debiting source and crediting destination. The Standing Instructions page displays a table with columns for Name, From Client, From Account, To Client, To Account, Amount, Validity (From/Till dates), and Status (Active, Disabled). A Create Standing Instruction button opens a form to collect Name (required), From Office/Client/Account Type/Account, To Office/Client/Account Type/Account, Transfer Type, Priority (dropdown), Instruction Type (Fixed or Dues), Amount (numeric), Validity From Date and Validity Till Date (date pickers), Recurrence Type (Periodic or As Per Dues), Recurrence Frequency (dropdown), and Recurrence Interval (numeric). Standing Instructions can be enabled, disabled, or deleted from the listing.

### 28. Accounting - Closures

The Closing Entries page is accessed from the Accounting menu. The page displays a table of existing closures with columns for Office, Closing Date, Created Date, and Comments. The Create Closure form collects Office (dropdown, required), Closing Date (date picker, required), and Comments (textarea). Creating a closure prevents journal entries from being posted for dates on or before the closing date, ensuring integrity of completed accounting periods. Only one closure per office-date combination permitted. Closures cannot be deleted once created but may be reversible per configuration. Closed periods block new loan/savings transaction creation on or before close date.

### 29. Tax Management

The Tax Management pages are accessed from the left sidebar under Products (Manage Tax Configurations). The Tax Components page displays a table with columns for Name, Percentage, Debit Account Type, Debit Account, Credit Account Type, Credit Account, and Start Date. The Create Tax Component form collects Name (required), Percentage (required numeric tax rate), Debit Account Type (dropdown: Asset, Liability, Equity, Income, Expense), Debit Account (GL account dropdown), Credit Account Type (dropdown), Credit Account (GL account dropdown), and Start Date (date picker, required). The Tax Groups page displays a table with Name and associated components. The Create Tax Group form collects Name (required) and Tax Components section allowing multiple components with Start/End dates. Tax groups combine multiple tax components into single rate. Tax groups are referenced by Savings Products (Enable Withhold Tax checkbox with Tax Group dropdown) and Charge definitions; when tax enabled on product, system applies configured tax rate on interest income or fees and posts corresponding journal entries to mapped GL accounts.

### 30. Organization Settings

The Organization section is accessed from the Admin menu. The Holidays page displays a table with Name, Start Date, To Date, Status (Active, Pending), and Rescheduling Type. The Create Holiday form collects Name (required), From/To Date (required, not future), Repayments Rescheduled To (date picker), Rescheduling Type (include/exclude/postpone), Description (textarea), and applicable Offices (multi-select). Holidays affect loan repayment schedules: include treats as regular days, exclude moves to next working day, postpone moves to manually specified date. The Working Days page allows configuring working days (Monday-Sunday checkboxes) and Repayment Rescheduling (Include Non-Working Days/Exclude/Reschedule to Working Day). The Currencies page allows selecting active currencies from global list (Code, Name, Decimal Places); at least one currency must remain active. The Funds page manages funding sources with Fund Name (required, unique) and External ID (optional). The Payment Types page lists configured methods (Cash, Bank Transfer, Mobile Money, Check, etc.) with Name, Description, Is Cash Payment indicator, and Position (sort order). The Bulk Import page allows importing multiple entity types (Clients, Groups, Centers, Offices, Staff, Users, Loans, Savings) with Download template and Upload interface; import history shows name, time, counts, and success/failure. The Global Configuration page displays system-wide feature flags (maker-checker 4-eyes approval, reschedule-future-repayments, allow-transactions-on-non-workingday, reschedule-repayments-on-holidays, savings-interest-posting-current-period-end, financial-year-beginning-month, meetings-mandatory-for-jlg-loans) with Name, Enabled toggle, Value, Description. The Manage Codes page lists code lists (Client Type, Gender, Address Type, Loan Purpose, etc.) allowing adding/editing/reordering/deactivating code values that populate system dropdowns. The Manage Data Tables page lists custom data tables extending core entities with custom fields, configurable with Data Table Name, Application Table Name (dropdown: m_client/m_group/m_loan/m_office/m_saving_account/m_product_loan/m_savings_product), Multi Row checkbox, and column definitions (Name, Type, Length, Is Mandatory, Is Unique).

### 31. System Administration

The Admin menu in the top navigation bar provides access to several system management pages. The Manage Scheduler Jobs page displays a table of all batch jobs with columns for Job Name, Is Active (toggle), CRON Expression, Previous Run Status (Success/Failed with times), Next Run Time, Currently Running indicator, and action buttons. Jobs include Apply Annual Fee, Add Accrual Transactions, Apply Penalty to Overdue Loans, Update Loan Summary, Post Interest for Savings, Transfer Fees, and Generate Loan Loss Provisioning. Each job can be toggled active/inactive and CRON expression edited. A global Start/Stop scheduler toggle controls whether all scheduled jobs execute. The Global Configuration page displays system-wide feature flags (maker-checker, reschedule-future-repayments, allow-transactions-on-non-workingday, reschedule-repayments-on-holidays, savings-interest-posting-current-period-end, financial-year-beginning-month, meetings-mandatory-for-jlg-loans) with Name, Enabled toggle, Value, and Description. The Manage Codes page lists all code lists (Client Type, Gender, Address Type, Loan Purpose, etc.) allowing adding, editing, reordering, and deactivating values. The Manage Data Tables page lists custom data tables extending core entities (m_client, m_group, m_loan, m_office, m_saving_account, m_product_loan, m_savings_product) with create form including Data Table Name, Application Table Name dropdown, Multi Row checkbox, and column definitions with Name, Type dropdown, Length, Is Mandatory, and Is Unique. The Audit Trails page displays all system actions with columns for Action Name, Entity Name, Resource ID, Maker, Made On Date, Checker, Checked On Date, Processing Result (Pending, Approved, Rejected), Office Name, and entity details. Filters allow searching by Action Name, Entity Name, Resource ID, Maker ID, Checker ID, Office, and Date Range. When maker-checker is enabled, pending commands appear with Approve and Reject buttons.

### 32. Logout

The logout function is accessible from the user profile icon in the top-right corner of the navigation bar. Clicking the icon reveals a dropdown menu with options for Profile Settings and Log Out. Clicking Log Out terminates the authenticated session, clears the authentication token stored in the browser, and redirects the user to the login page. After logout, any attempt to navigate to an authenticated page redirects to the login page.
